# Contributing to Elucid 🔦

Thank you for your interest in making Elucid better! This project is built for everyone, and we value contributions from users of all technical levels.

## How you can help

### 1. Add Examples
The heart of Elucid is its examples. If you find a topic where Claude's default response is fluff-heavy or confusing, and you've found a way to make it educational and direct using Elucid's principles, please add it to `references/examples.md`.

- **Goal:** Clear "Before" (default Claude) and "After" (Elucid-style) comparisons.
- **Format:** Follow the existing structure in `references/examples.md`.

### 2. Test the Skill
Install Elucid and use it in your daily conversations. 
- Does it explain jargon well? 
- Does it ever get too wordy? 
- Does it miss a chance to teach?
- Report issues or suggest rule tweaks in `SKILL.md`.

### 3. Translate the README
We want Elucid to be accessible to everyone. If you're fluent in another language, help us translate the `README.md` so more people can use it.

### 4. Improve the Rules
If you notice a recurring "bad habit" in Claude that Elucid doesn't catch yet, suggest a new rule or an update to an existing one in `SKILL.md`.

## Submission Process

1. **Fork** the repository.
2. **Create a branch** for your changes.
3. **Make your changes** (keep them focused and small).
4. **Submit a Pull Request** with a clear description of what you've added or changed.

## Community & Conduct

Be kind, be helpful, and remember that we're all here to learn.
